## Overview

The Proof Engine is TraceBurn's primary differentiator. Where Aura shows a number ("237 opt-outs sent") with no supporting evidence, TraceBurn generates a tamper-evident, timestamped, encrypted audit trail for every removal action. Every opt-out submission has receipts.

This document specifies the data model, capture pipeline, post-removal verification scheduler, export format, and activity log interface.

## 1. Proof Record Schema

### Pydantic Model

```python
from __future__ import annotations

import hashlib
from datetime import datetime
from enum import Enum
from pathlib import Path
from typing import Optional
from pydantic import BaseModel, Field, field_validator


class RemovalMethod(str, Enum):
    PLAYWRIGHT = "playwright"   # Tier 1: automated browser form submission
    EMAIL = "email"             # Tier 2: email opt-out
    MANUAL = "manual"           # Tier 3: user-guided manual submission


class ProofStatus(str, Enum):
    PENDING_VERIFICATION = "pending_verification"
    VERIFIED_ABSENT = "verified_absent"
    REAPPEARED = "reappeared"
    VERIFICATION_FAILED = "verification_failed"


class VerificationCheckpoint(BaseModel):
    """Result of a single post-removal re-verification pass."""
    checked_at: datetime                          # UTC ISO 8601
    status: ProofStatus
    http_response_code: Optional[int] = None
    note: Optional[str] = None                   # Plain language outcome description


class ProofRecord(BaseModel):
    """
    Immutable audit record for a single removal action.
    Written once at time of submission; verification checkpoints appended.
    """

    # Identity
    proof_id: str = Field(
        description="UUID4 — unique identifier for this proof record"
    )
    removal_request_id: int = Field(
        description="FK to removal_requests.id in SQLite"
    )

    # Broker context
    broker_name: str = Field(
        description="Human-readable broker name, e.g. 'Spokeo'"
    )
    broker_domain: str = Field(
        description="Canonical domain, e.g. 'spokeo.com'"
    )

    # Action details
    opt_out_url: str = Field(
        description="Exact URL used for the opt-out submission"
    )
    method: RemovalMethod = Field(
        description="Mechanism used to submit the removal request"
    )
    submitted_at: datetime = Field(
        description="UTC timestamp (ISO 8601) of the moment the opt-out was submitted"
    )
    http_response_code: Optional[int] = Field(
        default=None,
        description="HTTP status code returned by the opt-out endpoint (Tier 1/2 only)"
    )

    # Evidence
    screenshot_path: Optional[str] = Field(
        default=None,
        description=(
            "Path to the encrypted screenshot file, relative to the proof store root. "
            "Format: proof/{broker_name}_{YYYYMMDD_HHMMSS}.png.enc"
        )
    )
    screenshot_checksum: Optional[str] = Field(
        default=None,
        description=(
            "SHA-256 hex digest of the raw (unencrypted) screenshot bytes, "
            "computed before encryption. Used to detect tampering after decryption."
        )
    )
    confirmation_text: Optional[str] = Field(
        default=None,
        max_length=500,
        description=(
            "First 500 characters of confirmation text extracted from the broker "
            "page after opt-out submission (e.g. 'Your request has been received')."
        )
    )

    # Post-removal verification schedule
    # Timestamps are set when the check is scheduled; fulfilled when run.
    verify_at_7d: Optional[datetime] = Field(
        default=None,
        description="Scheduled UTC datetime for T+7 re-verification pass"
    )
    verify_at_30d: Optional[datetime] = Field(
        default=None,
        description="Scheduled UTC datetime for T+30 re-verification pass"
    )
    verify_at_90d: Optional[datetime] = Field(
        default=None,
        description="Scheduled UTC datetime for T+90 re-verification pass"
    )

    # Verification results (populated by the reverification scanner)
    checkpoint_7d: Optional[VerificationCheckpoint] = None
    checkpoint_30d: Optional[VerificationCheckpoint] = None
    checkpoint_90d: Optional[VerificationCheckpoint] = None

    # Derived status (denormalized for fast display)
    current_status: ProofStatus = ProofStatus.PENDING_VERIFICATION

    @field_validator("screenshot_checksum")
    @classmethod
    def checksum_must_be_sha256_hex(cls, v: Optional[str]) -> Optional[str]:
        if v is not None and len(v) != 64:
            raise ValueError("screenshot_checksum must be a 64-character SHA-256 hex digest")
        return v

    @field_validator("opt_out_url")
    @classmethod
    def url_must_be_https(cls, v: str) -> str:
        if not v.startswith("https://"):
            raise ValueError("opt_out_url must use HTTPS")
        return v

    class Config:
        use_enum_values = True
        json_encoders = {
            datetime: lambda dt: dt.strftime("%Y-%m-%dT%H:%M:%SZ")
        }
```

### SQLite Persistence

The `proof_records` table (defined in 1.5) stores the scalar fields. The `VerificationCheckpoint` sub-objects are stored as JSON blobs in `checkpoint_7d`, `checkpoint_30d`, `checkpoint_90d` columns and deserialized at read time. The `ProofRecord` model serves as the authoritative schema for both SQLite persistence and JSON export.

## 2. Screenshot Capture Design (Playwright)

### Principles

1. The unencrypted screenshot exists **only in memory** — never written to disk in plaintext.
2. The SHA-256 checksum is computed **before** encryption, over the raw bytes, providing tamper-evidence after decryption.
3. Encryption uses AES-256-GCM with the user's vault key (derived from their passphrase via PBKDF2HMAC — see 1.5 Architecture).
4. The encrypted file is written atomically using a temp-file-then-rename pattern to prevent partial writes.

### File Naming Convention

```
{proof_store_root}/proof/{broker_name}_{YYYYMMDD_HHMMSS}.png.enc
```

Examples:
```
~/.traceburn/proof/spokeo_20240315_094217.png.enc
~/.traceburn/proof/whitepages_20240315_101843.png.enc
~/.traceburn/proof/beenverified_20240315_110055.png.enc
```

`{broker_name}` is normalized: lowercase, spaces replaced with underscores, non-alphanumeric characters stripped.

### Capture Pipeline

```python
# src/proof/capture.py

import hashlib
import os
import tempfile
from datetime import datetime, timezone
from pathlib import Path

from cryptography.hazmat.primitives.ciphers.aead import AESGCM


def capture_and_store_screenshot(
    page,                    # playwright.sync_api.Page
    broker_name: str,
    proof_store_root: Path,
    vault_key: bytes,        # 32-byte AES-256 key, derived from vault passphrase
) -> tuple[str, str]:
    """
    Capture a full-page screenshot, compute its checksum, encrypt it, and
    write the encrypted file to the proof store.

    Returns:
        (screenshot_path_relative, sha256_hex_checksum)

    The unencrypted screenshot bytes are never written to disk.
    """

    # Step 1: Capture full-page screenshot into memory only
    raw_png_bytes: bytes = page.screenshot(full_page=True)

    # Step 2: Compute SHA-256 of raw bytes BEFORE encryption
    checksum = hashlib.sha256(raw_png_bytes).hexdigest()

    # Step 3: Encrypt with AES-256-GCM
    nonce = os.urandom(12)                          # 96-bit nonce, unique per file
    aesgcm = AESGCM(vault_key)
    encrypted_bytes = aesgcm.encrypt(nonce, raw_png_bytes, associated_data=None)
    # Stored format: [12-byte nonce][ciphertext+16-byte GCM tag]
    stored_bytes = nonce + encrypted_bytes

    # Step 4: Build file path
    timestamp_str = datetime.now(timezone.utc).strftime("%Y%m%d_%H%M%S")
    safe_broker = broker_name.lower().replace(" ", "_")
    filename = f"{safe_broker}_{timestamp_str}.png.enc"
    proof_dir = proof_store_root / "proof"
    proof_dir.mkdir(parents=True, exist_ok=True)
    target_path = proof_dir / filename

    # Step 5: Write atomically (temp file + rename)
    tmp_fd, tmp_path = tempfile.mkstemp(dir=proof_dir, suffix=".tmp")
    try:
        os.write(tmp_fd, stored_bytes)
        os.close(tmp_fd)
        os.replace(tmp_path, target_path)
    except Exception:
        os.unlink(tmp_path)
        raise
    finally:
        # Overwrite raw bytes in memory (best-effort, CPython GC is not guaranteed)
        raw_png_bytes = b"\x00" * len(raw_png_bytes)

    relative_path = str(Path("proof") / filename)
    return relative_path, checksum


def verify_screenshot_checksum(
    encrypted_path: Path,
    expected_checksum: str,
    vault_key: bytes,
) -> bool:
    """
    Decrypt a stored screenshot and verify its SHA-256 checksum.
    Returns True if the checksum matches (file unmodified since capture).
    """
    raw_data = encrypted_path.read_bytes()
    nonce = raw_data[:12]
    ciphertext = raw_data[12:]
    aesgcm = AESGCM(vault_key)
    decrypted = aesgcm.decrypt(nonce, ciphertext, associated_data=None)
    actual_checksum = hashlib.sha256(decrypted).hexdigest()
    return actual_checksum == expected_checksum
```

## 3. Post-Removal Verification Scanner

### Verification Logic

At each scheduled checkpoint (T+7, T+30, T+90), the verifier re-scans the broker using the same search terms that originally found the exposure. It compares the result against the original `exposure` record:

| Outcome | Action |
|---------|--------|
| Profile absent | Set `ProofStatus.VERIFIED_ABSENT`, update checkpoint timestamp, log `VERIFIED ABSENT` |
| Profile still present (initial check) | Log delay; reschedule 48h retry before escalating |
| Profile reappeared (after prior VERIFIED_ABSENT) | Set `ProofStatus.REAPPEARED`, set `exposure.status = "reappeared"`, enqueue re-removal, fire Telegram ALERT |
| Broker site unreachable | Set `ProofStatus.VERIFICATION_FAILED`, log error, reschedule 24h retry |

### APScheduler Job Design

```python
# src/proof/verifier.py

from apscheduler.schedulers.background import BackgroundScheduler
from apscheduler.jobstores.sqlalchemy import SQLAlchemyJobStore
from datetime import datetime, timezone

JOBSTORE_URL = "sqlite:///~/.traceburn/scheduler.db"

def build_scheduler() -> BackgroundScheduler:
    """
    Persistent job store backed by SQLite so scheduled checks survive
    process restarts. Jobs are recovered automatically on startup.
    """
    jobstores = {
        "default": SQLAlchemyJobStore(url=JOBSTORE_URL)
    }
    scheduler = BackgroundScheduler(jobstores=jobstores, timezone="UTC")
    return scheduler


def schedule_verification_checks(
    scheduler: BackgroundScheduler,
    proof_record: "ProofRecord",
) -> None:
    """
    Schedule T+7, T+30, and T+90 reverification jobs for a proof record.
    Each job runs run_verification_check() at the specified UTC datetime.
    """
    from datetime import timedelta

    base = proof_record.submitted_at
    checkpoints = [
        (7,  base + timedelta(days=7),  "7d"),
        (30, base + timedelta(days=30), "30d"),
        (90, base + timedelta(days=90), "90d"),
    ]

    for days, run_at, label in checkpoints:
        scheduler.add_job(
            func=run_verification_check,
            trigger="date",
            run_date=run_at,
            args=[proof_record.proof_id, label],
            id=f"verify_{proof_record.proof_id}_{label}",
            replace_existing=True,
            misfire_grace_time=3600,   # Run up to 1h late if process was down
        )


def run_verification_check(proof_id: str, checkpoint_label: str) -> None:
    """
    APScheduler calls this function at the scheduled time.
    Re-scans the broker and updates the proof record and exposure status.
    """
    from src.db import get_db
    from src.scanner.engine import spot_check_broker
    from src.notifier import send_alert

    db = get_db()
    proof = db.get_proof_record(proof_id)
    exposure = db.get_exposure(proof.removal_request_id)
    broker = db.get_broker(exposure.broker_id)
    user_profile = db.get_user(exposure.user_id)

    result = spot_check_broker(broker=broker, user_profile=user_profile)

    checkpoint = VerificationCheckpoint(
        checked_at=datetime.now(timezone.utc),
        http_response_code=result.http_code,
    )

    if result.profile_found and proof.current_status == ProofStatus.VERIFIED_ABSENT:
        # Data reappeared after a confirmed removal — escalate
        checkpoint.status = ProofStatus.REAPPEARED
        checkpoint.note = (
            f"Profile reappeared on {broker.name}. "
            f"Re-removal initiated automatically."
        )
        db.update_exposure_status(exposure.id, "reappeared")
        db.enqueue_removal(exposure.id)
        send_alert(
            level="ALERT",
            message=(
                f"Your data reappeared on {broker.name}. "
                f"TraceBurn has initiated a new removal. "
                f"Run `traceburn status` for details."
            )
        )

    elif result.profile_found:
        # Still present at first check — not yet escalating, just logging
        checkpoint.status = ProofStatus.PENDING_VERIFICATION
        checkpoint.note = f"Profile still present at {checkpoint_label} check. Monitoring."

    elif not result.profile_found and not result.site_reachable:
        checkpoint.status = ProofStatus.VERIFICATION_FAILED
        checkpoint.note = f"{broker.name} was unreachable during {checkpoint_label} check. Will retry."

    else:
        # Absent and site reachable — confirmed removal
        checkpoint.status = ProofStatus.VERIFIED_ABSENT
        checkpoint.note = f"Profile confirmed absent from {broker.name} at {checkpoint_label} check."
        if proof.current_status != ProofStatus.VERIFIED_ABSENT:
            db.update_exposure_status(exposure.id, "removed")

    db.save_verification_checkpoint(proof_id, checkpoint_label, checkpoint)
    db.update_proof_status(proof_id, checkpoint.status)
```

### Scheduler Lifecycle

The scheduler runs as a background thread when any `traceburn` command starts, using a persistent SQLite job store. This means:

- Scheduled jobs survive process restarts (job definitions are persisted).
- If `traceburn` is not running when a job fires, it runs within the `misfire_grace_time` window (1 hour) on the next invocation.
- Users who do not run TraceBurn for 90+ days will see pending verification jobs run on their next invocation.
- No daemon process or cron entry is required — the scheduler is embedded in the CLI process.

### PDF Export Content

Generated file: `proof_spokeo_2024-03-15.pdf`

PDF structure (top to bottom):

1. **Header** — TraceBurn wordmark + "Removal Proof Record" title + generation timestamp
2. **Broker details** — Name, domain, opt-out URL, removal tier
3. **Action record** — Method used, submission timestamp (UTC ISO 8601), HTTP response code
4. **Screenshot thumbnail** — Decrypted and embedded at 70% page width; below it: "Full-page screenshot captured at moment of form submission"
5. **Confirmation text** — Verbatim first 500 characters extracted from page, monospace block
6. **Tamper-evidence block** — SHA-256 checksum of screenshot, instructions for independent verification
7. **Verification history** — Table of T+7 / T+30 / T+90 checkpoints with status and timestamp
8. **Legal disclaimer** — Standard TraceBurn removal outcomes disclaimer (from 1.4)

### PDF Library Choice: WeasyPrint

**Selected: WeasyPrint** over reportlab.

| Criterion | WeasyPrint | reportlab |
|-----------|-----------|-----------|
| Template language | HTML + CSS (Jinja2) | Python API (imperative) |
| Maintainability | High — templates are readable HTML files editable without Python knowledge | Low — layout defined in code, hard to iterate |
| Layout flexibility | CSS flexbox/grid for clean typographic layout | Manual coordinate positioning |
| Image embedding | Native `<img>` tag with base64 | Requires `drawImage()` with explicit coordinates |
| Unicode/international | First-class via CSS font-face | Limited; requires manual font loading |
| Dependency | Pure Python + Pango/Cairo system libs | Pure Python |
| Verdict | ✅ Selected | ❌ Rejected |

**Rationale:** TraceBurn's proof PDFs must be legible, professional, and maintainable. WeasyPrint's HTML+CSS pipeline means the template lives in `src/reporting/templates/proof.html.j2` — readable, version-controlled, and editable without touching Python. reportlab's imperative API makes layout changes brittle and hard to review.

**Packaging note:** WeasyPrint requires `libpango`, `libcairo`, and `libgdk-pixbuf` system libraries. The `install.sh` installer checks for these and installs them via `apt`/`brew` as appropriate. For systems where this is impractical, the JSON export is always available without any system dependencies.

### Proof Export Implementation Sketch

```python
# src/reporting/generator.py

from pathlib import Path
from datetime import datetime, timezone
import base64
import json

from jinja2 import Environment, FileSystemLoader
from weasyprint import HTML


TEMPLATE_DIR = Path(__file__).parent / "templates"


def export_proof_pdf(
    proof: "ProofRecord",
    vault_key: bytes,
    output_path: Path,
    proof_store_root: Path,
) -> Path:
    """
    Render a WeasyPrint PDF for a single proof record.
    Decrypts the screenshot in memory for embedding; never writes it to disk.
    """
    from src.proof.capture import verify_screenshot_checksum
    from src.vault import decrypt_screenshot

    screenshot_b64 = None
    if proof.screenshot_path:
        enc_path = proof_store_root / proof.screenshot_path
        raw_bytes = decrypt_screenshot(enc_path, vault_key)
        # Verify integrity before embedding
        if not verify_screenshot_checksum(enc_path, proof.screenshot_checksum, vault_key):
            raise ValueError(f"Screenshot checksum mismatch for proof {proof.proof_id} — file may be corrupted")
        screenshot_b64 = base64.b64encode(raw_bytes).decode("ascii")

    env = Environment(loader=FileSystemLoader(str(TEMPLATE_DIR)))
    template = env.get_template("proof.html.j2")
    html_content = template.render(
        proof=proof,
        screenshot_b64=screenshot_b64,
        generated_at=datetime.now(timezone.utc).strftime("%Y-%m-%dT%H:%M:%SZ"),
    )

    HTML(string=html_content).write_pdf(str(output_path))
    return output_path


def export_proof_json(proof: "ProofRecord", output_path: Path) -> Path:
    with open(output_path, "w") as f:
        json.dump(proof.model_dump(mode="json"), f, indent=2, default=str)
    return output_path
```

### Filter Implementation

Filters are applied as SQL WHERE clauses on the `activity_log` table (joined with `brokers` and `proof_records`):

```python
# traceburn history --filter broker=spokeo --since 2024-01-01

def build_history_query(
    broker: Optional[str] = None,
    status: Optional[str] = None,
    since: Optional[datetime] = None,
) -> tuple[str, list]:
    conditions = []
    params = []
    if broker:
        conditions.append("LOWER(b.name) = LOWER(?)")
        params.append(broker)
    if status:
        conditions.append("al.status_label = ?")
        params.append(status)
    if since:
        conditions.append("al.event_at >= ?")
        params.append(since.isoformat())
    where = ("WHERE " + " AND ".join(conditions)) if conditions else ""
    sql = f"""
        SELECT al.event_at, al.action_label, b.name, al.status_label, al.proof_id
        FROM activity_log al
        LEFT JOIN brokers b ON al.broker_id = b.id
        {where}
        ORDER BY al.event_at DESC
        LIMIT 500
    """
    return sql, params
```

## Design Rationale Summary

| Decision | Choice | Why |
|----------|--------|-----|
| Screenshot encryption | AES-256-GCM with vault key | Consistent with vault encryption; GCM provides authenticated encryption (detects bit-flip tampering) |
| Checksum timing | Before encryption | Checksum over plaintext bytes means verification requires decryption — binding tamper-evidence to vault access |
| Atomic file write | tempfile + os.replace | Prevents partial writes from leaving a corrupt `.enc` file if the process is killed mid-write |
| Job store | APScheduler + SQLiteJobStore | Survives process restarts; no daemon required; matches project's embedded-SQLite philosophy |
| PDF library | WeasyPrint | HTML+CSS template is maintainable without Python expertise; international text support via Pango |
| JSON export | Always available | WeasyPrint has system library dependencies; JSON requires none |
| Confirmation text limit | 500 chars | Captures the meaningful part of the confirmation message; prevents bloating proof records with full page HTML |
