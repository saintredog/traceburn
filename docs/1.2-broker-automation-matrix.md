## Summary (Key Findings)

- **5 brokers** support full or near-full automation (web form + email confirmation automatable end-to-end).
- **6 brokers** are partially automatable — the form can be scripted but a manual step (phone verification, CAPTCHA, or ID upload) blocks full automation.
- **9 brokers** require fully manual steps (written requests, phone calls, identity verification, or legal process).
- The biggest automation blockers are: **CAPTCHA**, **phone/SMS verification**, and **government ID requirements**.
- The **PeopleConnect network** (BeenVerified, TruthFinder, InstantCheckmate, PeopleLooker, USSearch, Intelius) may share opt-out propagation — one submission could clear multiple listings. This should be tested during Phase 4 live testing.
- Removing from **Acxiom** is the highest-leverage action among enterprise brokers — many smaller brokers source data from Acxiom, so removal there may reduce re-listing rates downstream.

## Automation Tiers — Detailed Breakdown

### Tier 1 — Fully Automatable (No Manual Steps)

These brokers can be handled end-to-end by a Playwright script with no human-in-the-loop required.

| Broker | Method | Notes |
|---|---|---|
| PeopleFinders | Playwright form submission | No email confirmation required in current flow |
| CheckPeople | Playwright form submission | Minimal friction; verify no CAPTCHA appears |

**Implementation approach:** Playwright script navigates to opt-out URL, fills form fields with PII vault data, submits. Log HTTP response status. Verify removal via re-scan 24h later.

---

### Tier 2 — Partially Automatable (Email Confirmation Required)

These brokers require clicking a link in a confirmation email. This can be automated by integrating IMAP access to a dedicated opt-out email address.

| Broker | Notes |
|---|---|
| Spokeo | Email confirmation link must be clicked |
| Intelius | Email confirmation link must be clicked |
| BeenVerified | Email confirmation link must be clicked |
| TruthFinder | Email confirmation link must be clicked |
| InstantCheckmate | Email confirmation link must be clicked |
| PeopleLooker | Email confirmation link must be clicked |

**Implementation approach:**
1. Playwright fills and submits opt-out form using a dedicated TraceBurn opt-out email address (e.g., `traceburn-optout@gmail.com`)
2. Python script polls inbox via IMAP (imaplib) for confirmation emails from each broker domain
3. Extract confirmation URL from email body (regex or BeautifulSoup)
4. Playwright or requests.get() opens the confirmation URL
5. Log success and timestamp

**Risk:** Some brokers may switch to SMS-based confirmation. Monitor for this during testing.

### Tier 3 — Manual Only (Human-in-the-Loop Required)

These brokers require a manual step that cannot be automated without significant additional infrastructure (CAPTCHA solving service, phone verification service, or ID scan submission).

| Broker | Blocker | Manual Process | Workaround (Future) |
|---|---|---|---|
| Whitepages | Phone/SMS verification | User must receive a phone call or text to verify | Twilio integration (use TraceBurn's own number) |
| FamilyTreeNow | CAPTCHA | User must solve CAPTCHA | 2captcha / Anti-CAPTCHA API service integration |
| USSearch | Phone verification | Same as Whitepages | Twilio integration |
| ZabaSearch | Written email required | Send email to privacyrequest@intelius.com | SMTP automation (lowest friction, already easy) |
| LexisNexis | Government ID upload | Must upload driver's license or passport scan | Cannot automate — flag for manual review queue |

**Recommended handling in TraceBurn:**
- Tier 3 brokers should populate a **Manual Queue** in the removal dashboard
- The removal engine should generate a pre-filled instructions guide for each manual opt-out (name, address to provide, exact URL to visit)
- LexisNexis should generate a reminder with a checklist since it requires physical ID

## CAPTCHA Strategy

CAPTCHAs are the primary automation blocker. Options:

| Approach | Pros | Cons |
|---|---|---|
| Manual fallback | No cost, no TOS risk | Defeats automation goal |
| 2captcha API | Automated, fast (~10–30s) | ~$1–3 per 1000 CAPTCHAs; requires API key |
| Anti-CAPTCHA API | Similar to 2captcha | Similar cost |
| hCaptcha accessibility | Some sites accept audio CAPTCHA | Unreliable |

**Recommendation:** Implement manual fallback first (Phase 3). Add 2captcha integration as optional enhancement in Phase 5 if CAPTCHA-blocked brokers are high-priority.

---

## Phone Verification Strategy

Whitepages and USSearch require phone verification. Options:

| Approach | Pros | Cons |
|---|---|---|
| Use real phone (guided) | Most reliable | Manual step required |
| Twilio number | Automatable (receive SMS/call) | Cost (~$1/mo per number); requires setup |
| Google Voice | Free | Less automatable; account required |

**Recommendation:** Use guided manual step with Twilio integration as Phase 5 enhancement.
